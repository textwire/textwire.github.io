# Version 4 Release Notes

Textwire 4 is a major release that introduces error handling improvements and consistency improvements. Most of the work was internal and will not effect the API, but there are still things you need to pay attention.

If you are transitioning from Textwire v3, you can follow [this upgrade guide](/v4/upgrade) for all the instructions. v4 contains several breaking changes, so make sure you don't miss any parts of the guide.

## New Error Return Type

Previously, there was no way to get detailed error information from Textwire, except `String()` function on `*Template` type. Go's standard errors just hold an error message, but our `*fail.Error` provides much more context about the error, making it easier to debug and understand what went wrong. With `*fail.Error`, you can access:

- **File path** with `Filepath()` method
- **Error position** with `Pos()` method
- **Error message** with `Message()` method
- **Convert to standard error** with `Error()` method

And other things things that will be important for LSP servers, libraries, and other tools that want to provide better error reporting.

```go
tpl, err := textwire.NewTemplate(nil)
if err != nil {
    fmt.Printf(
        "Error in %s at line %d, col %d: %s\n",
        err.Filepath(),
        err.Pos().StartLine,
        err.Pos().StartCol,
    )
}
```

See the the [Upgrade Guilde](/v4/upgrade#_2-error-handling-changes) for this specific change.

## Postfix

Since this is a templating engine for Go, the increment/decrement statements should also behave the same way.
