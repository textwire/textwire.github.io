# Version 4 Release Notes


If you are transitioning from Textwire v3, you can follow [this guide](/v4/upgrade) for all the instructions. v4 contains several breaking changes, so make sure you don't miss any parts of the guide.

## Breaking Changes

#### Why `*fail.Error`?
Unlike standard Go errors, `*fail.Error` provides detailed context for debugging:
- **Line and column positions** — `LineStart`, `LineEnd`, `ColStart`, `ColEnd`
- **File path** — Know exactly which template file has the issue
- **Detailed messages** — Clear explanations of what went wrong
Example:
```go
tpl, err := textwire.NewTemplate(nil)
if err != nil {
    // Access rich error information
    fmt.Printf("Error in %s at line %d, col %d: %s\n",
        err.Filepath, err.LineStart, err.ColStart, err.Message)
}
```

## Postfix

Since this is a templating engine for Go, the increment/decrement statements should also behave the same way.
